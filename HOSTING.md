# Инструкция по размещению сайта на хостинге

## Размещение на Рег.ру

### Шаг 1: Подготовка файлов

1. Откройте проект в poehali.dev
2. Нажмите **Скачать → Скачать билд**
3. Распакуйте скачанный архив
4. Внутри найдете папку `dist` со всеми файлами сайта

### Шаг 2: Загрузка файлов на хостинг

1. Войдите в панель управления хостингом на сайте Рег.ру
2. Перейдите в **Файловый менеджер** или используйте FTP-клиент (FileZilla, WinSCP)
3. Найдите корневую директорию сайта (обычно `public_html` или `www`)
4. Загрузите **содержимое папки dist** (не саму папку!) в корневую директорию

### Шаг 3: Настройка React Router (важно!)

Чтобы все страницы сайта открывались корректно, создайте файл `.htaccess` в корне сайта:

1. В корневой папке создайте новый файл с именем `.htaccess`
2. Вставьте в него следующий код:

```apache
<IfModule mod_rewrite.c>
  RewriteEngine On
  RewriteBase /
  RewriteRule ^index\.html$ - [L]
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteRule . /index.html [L]
</IfModule>
```

3. Сохраните файл

### Шаг 4: Проверка

Откройте ваш домен в браузере. Сайт должен работать! Проверьте:
- Главную страницу
- Страницу каталога
- Корзину
- Админ-панель

## Размещение на других хостингах

### GitHub Pages

1. Создайте репозиторий на GitHub
2. Загрузите содержимое папки `dist`
3. В настройках репозитория включите GitHub Pages
4. Выберите ветку `main` и папку `/ (root)`

### Netlify

1. Перетащите папку `dist` на [netlify.com/drop](https://app.netlify.com/drop)
2. Сайт опубликуется автоматически
3. Для настройки маршрутов создайте файл `_redirects` в папке `dist`:
```
/*  /index.html  200
```

### Vercel

1. Установите Vercel CLI: `npm i -g vercel`
2. В папке с проектом выполните: `vercel --prod`
3. Следуйте инструкциям в терминале

## Часто задаваемые вопросы

**Q: Почему при открытии index.html локально белый экран?**

A: React-приложения требуют веб-сервер для работы. Используйте команду `npx serve dist` для локального просмотра.

**Q: Страницы не открываются напрямую (404 ошибка)**

A: Нужно настроить редирект на index.html через `.htaccess` (для Apache) или конфигурацию сервера.

**Q: API не работает после загрузки на хостинг**

A: Убедитесь, что в коде используются правильные URL к backend-функциям. Все функции работают через URLs из `func2url.json`.

## Нужна помощь?

Если возникли проблемы с размещением сайта:
1. Проверьте логи хостинга
2. Убедитесь, что все файлы загружены корректно
3. Проверьте, что `.htaccess` создан правильно
4. Обратитесь в поддержку вашего хостинг-провайдера
